## Problem

Therapist federated feed (SCDF/SCID-5/MSHE) shows "Consentimiento federado requerido" even when the client agreed verbally, because `Patient.consent_federation` stayed `False` with no UI or API to set it.

## Solution

**Part A — immediate unblock (prod):**
```bash
docker exec studio33_api python manage.py grant_federation_consent --patient-id 1
```

**Part B — sustainable flow:**
- `GET/POST /api/patient/federation-consent/` — client portal toggle (Mi cuenta)
- `POST /api/therapist/patients/{id}/federation-consent/` — therapist records in-session consent
- Append-only `FederationAuditLog` entries on each change (no model/schema changes)
- `FederationHubFeedBlock`: "Registrar consentimiento en sesión" button reloads feed

## Tests

`api.tests.test_federation_consent`:
- (a) no consent → feed 403
- (b) grant via patient portal → feed 200
- (c) revoke → feed 403 again
- therapist in-person grant + idempotent grant
