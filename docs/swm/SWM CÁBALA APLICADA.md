## 1. Propósito del SWM Cábala Aplicada

El SWM Cábala Aplicada es un **entorno de exploración simbólica profunda**, diseñado para acompañar procesos de conciencia, sentido, integración del alma y comprensión de patrones espirituales **sin carácter clínico ni terapéutico directo**.

Su función es:

- Facilitar comprensión simbólica del ser humano
- Explorar el Árbol de la Vida como mapa de conciencia
- Integrar lenguajes cabalísticos tradicionales de forma accesible
- Acompañar (no dirigir) procesos humanos y terapéuticos

Este SWM **no diagnostica**, **no evalúa**, **no concluye**.

---

## 2. Qué ES y qué NO es este SWM

### ES
- Un workspace independiente
- Un entorno de sabiduría estructural
- Un sistema de mapas simbólicos
- Un espacio de observación, estudio y reflexión aplicada

### NO ES
- Un módulo clínico
- Un panel auxiliar
- Un sistema predictivo
- Un generador de verdades espirituales
- Un sustituto del terapeuta o guía humano

---

## 3. Alcance funcional del SWM

El SWM Cábala Aplicada **contiene internamente** (no como SWM separados):

### 3.1 Núcleo estructural
- Árbol de la Vida (Sefirot)
- Senderos
- Planos de conciencia
- Estructura alma–mundo–acción

### 3.2 Lenguajes cabalísticos
- Gematría
- Notarikón
- Temurá
- Nombres y combinaciones
- Letras hebreas (como símbolos, no talismanes)

### 3.3 Aplicaciones internas
- Mapas del alma
- Ciclos de tikún
- Correspondencias simbólicas
- Lecturas integrativas (no deterministas)

👉 **Nada de esto es un módulo separado**.  
Todo vive **dentro del mismo SWM**.

---

## 4. Relación con el Core Workspace

- El SWM se abre desde el contexto terapéutico
- Hereda paciente, sesión y rol (solo lectura)
- No altera el workspace clínico central
- Permite retorno explícito al Core

Nunca sustituye ni degrada el Core.

---

## 5. Aislamiento funcional (obligatorio)

El SWM Cábala Aplicada:

🚫 No escribe en registros clínicos  
🚫 No ejecuta scoring  
🚫 No genera diagnósticos  
🚫 No produce conclusiones cerradas  
🚫 No automatiza decisiones  

Toda información generada es:
- consultiva
- simbólica
- humana
- contextual

---

## 6. Estructura mínima del Workspace

### 6.1 Layout propio
- Header contextual
- Área central (mapa simbólico)
- Sidebar interno de herramientas
- Paneles internos no modales

### 6.2 Núcleo visual
El Árbol de la Vida es el **centro visual persistente**.

- Nunca se oculta automáticamente
- No reacciona solo
- No emite significado por sí mismo

---

## 7. Principios UX del SWM Cábala

### UX-01 — Neutralidad simbólica  
Los símbolos no indican “bien/mal”, “equilibrado/desequilibrado”.

### UX-02 — No determinismo  
El sistema nunca afirma destino, causa ni resultado.

### UX-03 — Activación consciente  
Nada se muestra sin acción humana explícita.

### UX-04 — Silencio visual  
El Árbol puede observarse sin que el sistema “hable”.

### UX-05 — Comprensión gradual  
El usuario decide profundidad y ritmo.
