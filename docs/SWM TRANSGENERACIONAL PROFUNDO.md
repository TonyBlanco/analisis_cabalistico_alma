## 1. Propósito del SWM Transgeneracional Profundo

El SWM Transgeneracional Profundo es un **entorno especializado de exploración del linaje humano**, diseñado para acompañar la comprensión de **patrones heredados, repeticiones, memorias familiares y dinámicas sistémicas**, desde una perspectiva:

- simbólica  
- experiencial  
- humana  
- no clínica  

Su objetivo es **hacer visible lo invisible del sistema familiar**, sin emitir diagnósticos ni conclusiones deterministas.

Este SWM **no evalúa**, **no diagnostica**, **no predice**.

---

## 2. Qué ES y qué NO es este SWM

### ES
- Un workspace independiente
- Un espacio de observación del sistema familiar
- Un contenedor de relatos, eventos y vínculos
- Un soporte para procesos de conciencia y reparación simbólica

### NO ES
- Un módulo clínico
- Un sistema psicológico formal
- Un generador automático de conflictos
- Un evaluador de familias
- Un sustituto del criterio humano

---

## 3. Alcance funcional del SWM

El SWM Transgeneracional Profundo contiene internamente:

### 3.1 Árbol genealógico vivo
- Representación gráfica del linaje
- Múltiples generaciones
- Líneas materna y paterna
- Relaciones explícitas (sin inferencia)

### 3.2 Eventos significativos
- Nacimientos
- Muertes
- Abortos / pérdidas gestacionales
- Migraciones
- Exclusiones
- Rupturas
- Enfermedades (como hechos narrativos, no diagnósticos)

### 3.3 Patrones y repeticiones
- Fechas
- Nombres
- Roles
- Destinos similares
- Lealtades visibles

⚠️ **Nada se genera automáticamente.**  
Todo se introduce, observa y valida de forma humana.

---

## 4. Relación con el Core Workspace

- El SWM hereda paciente, sesión y rol (solo lectura)
- No modifica el estado clínico
- No escribe en historiales oficiales
- Permite retorno explícito al Core

El trabajo transgeneracional **no sustituye** el proceso terapéutico central.

---

## 5. Aislamiento funcional obligatorio

Está prohibido en este SWM:

🚫 Diagnóstico psicológico  
🚫 Etiquetado familiar  
🚫 Inferencia automática  
🚫 Scoring  
🚫 Conclusiones cerradas  

Toda información es:
- narrativa
- contextual
- editable
- no normativa

---

## 6. Estructura mínima del Workspace

### 6.1 Layout propio
- Header contextual del linaje
- Área central: Árbol genealógico
- Sidebar interno de herramientas
- Paneles internos no modales

### 6.2 Núcleo visual
El **árbol genealógico** es el centro visual persistente.

- Nunca se oculta automáticamente
- No se sustituye por paneles
- Es el punto de referencia continuo

---

## 7. Principios UX del SWM Transgeneracional

### UX-01 — No inferencia
El sistema nunca deduce vínculos, traumas o conflictos.

### UX-02 — Neutralidad narrativa
Los eventos se muestran sin jerarquía moral.

### UX-03 — Tiempo humano
El terapeuta decide ritmo, profundidad y foco.

### UX-04 — Visibilidad sin dramatización
No hay colores de alarma ni iconografía de urgencia.

### UX-05 — Silencio sistémico
El árbol puede observarse sin intervención del sistema.

---

## 8. Sexo biológico, género y linaje

### 8.1 Sexo biológico
Se registra como dato factual cuando es relevante para:
- maternidad
- paternidad
- abortos
- filiación biológica

### 8.2 Identidad de género
- Se respeta
- Se documenta
- No altera la estructura genealógica

El árbol representa **hechos relacionales**, no identidades normativas.

## 10. Marco ético fundamental

Principios inmutables:

- El linaje no es culpable
- La herencia no es condena
- La repetición no es destino
- La conciencia abre opciones

El sistema **acompaña**, no sentencia.

---

## 11. Relación con IA (futuro)

La IA podrá:

✔ Ayudar a resumir historias  
✔ Detectar repeticiones narrativas  
✔ Organizar eventos cronológicos  

La IA **no podrá**:

🚫 Interpretar traumas  
🚫 Definir conflictos  
🚫 Etiquetar relaciones  
🚫 Concluir dinámicas  

Lenguaje obligatorio:
- “posibles repeticiones observadas”
- “patrones narrativos”
- “hipótesis no concluyentes”
